# OSF deposit plan

The protocol's Data availability statement commits to depositing all analysis code on
the Open Science Framework (OSF) prior to publication. This plan keeps that promise
concrete.

## What gets deposited
- The R pipeline (`06_analysis/R/`) and the analysis-ready `meta_input.csv`.
- Search strategies (`02_search/`), screening criteria, extraction codebook, RoB and
  GRADE templates.
- The registered protocol + PRISMA materials.
- Forest/funnel/RoB figures generated into `06_analysis/results/` (these are git-ignored
  here but are deposit artefacts).

## What does NOT get deposited
- No primary patient data (none are generated by this review).
- Copyrighted full texts of included studies.

## Steps (at/just before submission)
1. Create an OSF project; link the PROSPERO record (CRD420261409919).
2. Push a tagged snapshot of `ST10_Remision_study/` (minus copyrighted PDFs).
3. Mint an OSF DOI; insert it into the manuscript Data availability statement.
4. If a public Git repo is used, cross-link OSF ↔ repo.
